# 📁 Observatório de Projetos Integradores


## Português

### Qual a ideia do projeto?
Criar um sistema web pra organizar os projetos integradores. Atualmente, os Projetos Integradores (PIs) são entregados por diferentes plataformas: e-mail, teams, etc. A ideia do projeto seria desenvolver uma aplicação para facilitar a vida dos alunos, professores e coordenadores

###  O que a gente quer com isso?
Deixar a vida de alunos, professores e coordenadores mais simples:
- Aluno sobe seu projeto e monta um portfólio legal.
- Professor avalia direto na plataforma.
- Coordenador acompanha tudo e gera relatórios.
- Empresa parceira pode olhar os projetos e encontrar talentos.

###  Tecnologias que vamos usar (por enquanto)
Ainda podemos mudar, mas o plano é:
- Backend: Python com Djongo
- Banco de dados: MySQL
- Frontend: HTML, CSS e JavaScript
- Versionamento: Git + GitHub
- Organização: Jira

###  Regras do negócio (as mais importantes)
- Só o coordenador pode criar usuários.
- O aluno mexe **só nos seus próprios projetos**.
- O professor vê os projetos da sua turma e avalia com rubrica.
- A empresa **só visualiza** – não mexe em nada.
- Todo mundo tem que seguir a LGPD (proteção de dados).
- Cada grupo precisa inventar **um diferencial criativo** no sistema.

### English
### What's the idea?

A web system to organize integrative projects. No more lost emails, wrong versions, or messy files. Everything in one place.

### What do we want?
Make life easier for students, teachers, and coordinators:

Students submit projects and build a portfolio.

Teachers grade projects inside the platform.

Coordinators see everything and generate reports.

Partner companies can browse projects and find new talent.

###  Technologies
We might change things, but the plan is:

Backend: Python + Djongo

Database: MySQL 

Frontend: HTML, CSS, and JavaScript

Version control: Git + GitHub

Organization: Jira

### Key business rules
Only the coordinator can create users.

Students can only edit their own projects.

Teachers see only their classes and grade with a rubric.

Companies can only view projects – no editing.

We must follow data protection laws (LGPD).

Each group must add a creative differential to the system.
